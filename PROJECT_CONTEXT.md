## 1. Project Overview

**ProfitPilot** (also referenced in docs as *Intelligent Business Agent* / *AI Business Helper*) is a product for **small business owners**: a marketing landing site, a **Next.js analytics dashboard**, and a **Flask + LangGraph** backend that answers natural-language questions via **SSE streaming**, with optional **PostgreSQL** business data, **Prometheus** metrics, and **Loki** logs. The vision (see `PS.md`) is an AI partner that understands business numbers, warns about risky decisions, and surfaces health-style insights—much of that is aspirational versus what is wired in code today.

```
  │       Google OAuth (client-side) → userinfo API (Google)

  │       /api/* ── rewrite ──► AGENT_API_URL (Docker: http://backend:5000)
  │       Dashboard charts call /api/dashboard/* on agent (partial coverage); many endpoints fall back to mocks in api.ts
  │
  ├── Legacy web/ Flask (:5001) — NOT in docker-compose
  │       /api/dashboard/* → real SQL from PostgreSQL (daily_transactions, alerts, …)
  │       /api/chat/send → proxies SSE from Agent :5000; SQLite chat history
  │
  ├── Grafana (:3000), Prometheus (:9090), Loki (:3100)
  │       Prometheus scrapes: nextjs-dashboard:3000/metrics, backend:5000/metrics
  │       Promtail → ships ./logs → Loki (job=python_app)
  │
  └── pgAdmin (:5050) → PostgreSQL
```

**LangGraph:** `agent_code/app.py` routes queries to subgraphs (general info, database, logs, metrics) with **PostgresSaver** checkpointers on `DATABASE_URL`.

**Ollama:** Expected at `LLM_BASE_URL` (default `http://127.0.0.1:11434/`). Not defined as a Docker service in this compose file.

## 6. Data Flow — Key Journeys

### 6.1 User Onboarding Flow
1. User opens landing `/get-started` (`get-started.tsx`).  
2. Form POST to **`http://localhost:5000/api/v1/onboarding`** (hardcoded).  
3. `agent_code/app.py` → `onboarding()` inserts into `public.businesses`, `public.roles`, `public.users`.  
4. **Risk:** SQL schema in `company_db_schema.sql` may not include all columns the INSERT expects (e.g. `city`, `monthly_revenue` string) — onboarding may fail until DB migrated.  
5. Client sets `localStorage` flags and redirects to `dashboardUrl` (`http://localhost:3001`).  

### 6.2 Chat / Query Flow (agent)
1. Client calls `GET /api/v1/query?input-query=...&thread-id=...` with **`Accept` SSE** (used by `web/app.py` proxy).  
2. `intent_detection.detect_intent` (Ollama).  
3. Branch: general (invoke + `format_response_stream`), database/logs/metrics (`_stream_graph` with LangGraph `stream`).  
4. SSE events: `status`, `token`, `clarification`, `final`, `error`.  

### 6.3 Dashboard Data Flow (Next.js)
1. Browser loads `dashboard` page; components call `api.get*` in `lib/api.ts`.  
2. Request goes to **same origin** `/api/dashboard/...` → Next **rewrite** → **`AGENT_API_URL`** (backend container).  
3. Agent implements only a **subset** of dashboard routes (synthetic data from `businesses`). Missing routes return error → **`fetchWithFallback` uses mocks**.  
4. For **full real SQL** dashboard APIs, run **`web/app.py` on 5001** and point clients there — not wired in compose.

## 10. How to Run Locally

### Docker (recommended)
```bash
cd intelligent-business-agent
# Ensure ./.env exists with at least DATABASE_URL pointing at db service, e.g.:
# DATABASE_URL=postgresql://admin:root@db:5432/test_db
docker compose up --build
```

- Landing: http://localhost:5173  
- Agent API: http://localhost:5000  
- Dashboard: http://localhost:3001  
- Postgres: localhost:5432  
- pgAdmin: http://localhost:5050  
- Grafana: http://localhost:3000  
- Prometheus: http://localhost:9090  
- Loki: http://localhost:3100  

Apply schema/data: follow `README.md` steps using `company_db_schema.sql` and `inserts.sql` (or reconcile onboarding columns first).

### Manual dev
```bash
# PostgreSQL (or use Docker only db)
# Agent — requires Ollama running at LLM_BASE_URL
cd agent_code
pip install -r requirements.txt
python app.py   # listens on 0.0.0.0:5000

# Legacy web dashboard + chat proxy
cd web
pip install -r requirements.txt
python app.py   # :5001

# Landing
cd landing-page
npm install
npm run dev -- --host 0.0.0.0 --port 5173

# Dashboard
cd dashboard
npm install
npm run dev   # default Next port 3000; use 3001 if needed: PORT=3001 npm run dev
# Set AGENT_API_URL=http://localhost:5000 for API rewrites
```
